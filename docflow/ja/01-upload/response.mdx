---
title: "レスポンス JSON 構造の説明"
---

<Tip>
  Docflow のファイルアップロードは非同期 API です。\
  API 呼び出しが成功すると、後続の結果取得に使用する結果識別子（`batch_number` と `ファイル ID`）が返されます。
</Tip>

アップロード API のリクエストが成功すると、次の形式で JSON レスポンスが返されます。

```json expandable
{
   "code":200,
   "msg":"成功",
   "result":{
      "batch_number":"1955840540571376304",
      "files":[
         {
            "id":"1955840505753140508",
            "task_id":"1981692246135111680",
            "name":"企业信用报告.pdf",
            "format":"pdf"
         }
      ]
   }
}
```

## 主要フィールドの説明

- `batch_number`: ファイルバッチ番号。同一バッチ内の複数ファイルに関連付けることができます。
- `files[].id`: 今回アップロードしたファイルの ID。
- `files[].task_id`: ファイル処理タスクを識別するためのタスク ID。

`batch_number`、`files[].id`、`files[].task_id` はいずれも後続の結果取得時の検索条件として使用できます。
