## 2026-07-05

### API

#### 抽出モデルのアップグレード

抽出モデルの名称を統一し、新たに **Auto** スマートルーティングを追加しました。`extract_model` を含むすべてのエンドポイント（**ファイルカテゴリを作成**、**ファイルカテゴリを更新**、ファイルカテゴリのフィールド/テーブルの追加と更新、**特定フィールドを抽出**、**ファイル処理結果一覧を取得**）が対象です。

| 旧モデル名 | 新モデル名 | 説明 |
|---------|---------|------|
| Model 1 | `Acgpt` | 高速で安定した抽出結果 |
| Model 2 | `DF-M1` | 複雑な文書理解に適しています |
| Model 3 | `Acgpt-VL` | VLM。シンプルな抽出に適しています（≤10 ページ） |
| — | `Auto` | 抽出モデルを自動でマッチング（新規追加。アルゴリズムがフィールドごとに実際のモデルを自動でルーティングします） |

- リクエストパラメータ `extract_model` は新モデル名（`Auto`/`Acgpt`/`Acgpt-VL`/`DF-M1`）のみを受け付けます。旧値 `Model 1`/`Model 2`/`Model 3` および過去の code 値 `llm`/`vlm` はサポートされなくなりました。
- **ファイル処理結果一覧を取得**、**特定フィールドを抽出** のレスポンスで、フィールドとテーブルに `configModel`（設定モデル。`Auto` を取り得ます。既存データで未設定の場合は `null`）を追加しました。従来の `extractModel` は「実際にヒットしたモデル」（値は `Acgpt`/`Acgpt-VL`/`DF-M1`）に意味を変更しました。
- 抽出失敗理由に「Acgpt-VL の処理上限を超過」を追加しました（`Acgpt-VL` には文書レベルの処理上限があり、ページ数超過時などに通知されます）。
- 抽出モデル `Acgpt-VL` は現在、テーブルの複数テーブル結合（`collect_from_multi_table`）に対応していません。

#### 1 回のリクエストでカテゴリを作成

[ファイルカテゴリを作成](/api-reference/%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%AB%E3%83%86%E3%82%B4%E3%83%AA/%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%AB%E3%83%86%E3%82%B4%E3%83%AA%E3%82%92%E4%BD%9C%E6%88%90) が `tables` パラメータに対応しました。カテゴリ作成時に、テーブルとそのフィールドを 1 回の呼び出しで設定できます。

#### フィールドの高度な設定

[ファイルカテゴリフィールドを更新](/api-reference/%E5%8D%98%E4%B8%80%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%AB%E3%83%86%E3%82%B4%E3%83%AA%E3%83%95%E3%82%A3%E3%83%BC%E3%83%AB%E3%83%89%E3%82%92%E6%9B%B4%E6%96%B0)（単一更新 + [一括更新](/api-reference/%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%AB%E3%83%86%E3%82%B4%E3%83%AA/%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%AB%E3%83%86%E3%82%B4%E3%83%AA%E3%81%AE%E3%83%95%E3%82%A3%E3%83%BC%E3%83%AB%E3%83%89%E3%82%92%E6%9B%B4%E6%96%B0)）が、以下のプロパティに対応しました。

| プロパティ | 説明 | 更新時の動作 |
|----------|-------------|-----------------|
| `alias` | フィールド別名 | 省略=変更なし、`[]`=クリア、`["a","b"]`=上書き |
| `identity` | エクスポート時のフィールド名 | 省略=変更なし、`""`=クリア、空でない値=上書き |
| `multi_value` | 複数値抽出 | 省略=変更なし、`true`/`false`=上書き |
| `duplicate_value_distinct` | 重複値の除去 | `multi_value=true` の場合のみ有効 |
| `transform_settings` | 出力変換設定 | datetime / enumerate / regex に対応 |

#### レスポンス情報の拡充

追加のクエリなしで完全な設定を取得できます。

- [ファイルカテゴリのフィールド一覧を取得](/api-reference/%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%AB%E3%83%86%E3%82%B4%E3%83%AA/%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%AB%E3%83%86%E3%82%B4%E3%83%AA%E3%81%AE%E3%83%95%E3%82%A3%E3%83%BC%E3%83%AB%E3%83%89%E4%B8%80%E8%A6%A7%E3%82%92%E5%8F%96%E5%BE%97) — `enabled`, `prompt`, `use_prompt`, `extract_model`, `alias`, `identity`, `multi_value`, `duplicate_value_distinct`, `transform_settings` を追加

- [ファイルカテゴリ一覧を取得](/api-reference/%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%AB%E3%83%86%E3%82%B4%E3%83%AA/%E3%83%AF%E3%83%BC%E3%82%AF%E3%82%B9%E3%83%9A%E3%83%BC%E3%82%B9%E3%81%AE%E3%83%95%E3%82%A1%E3%82%A4%E3%83%AB%E3%82%AB%E3%83%86%E3%82%B4%E3%83%AA%E4%B8%80%E8%A6%A7%E3%82%92%E5%8F%96%E5%BE%97) — `category_prompt`, `extract_model` を追加
